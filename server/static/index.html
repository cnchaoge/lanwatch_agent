核心能力

从“发现异常”到“判断原因”，把网络值守做成一套完整流程。

LANWatch 不只是一个在线率面板，而是一套面向中小企业网络场景的轻量监控产品。重点不是展示更多图表，而是在故障发生时更快给出可执行判断。

🔍

离线断点诊断

当客户端连续失败达到阈值后，自动触发 traceroute 并记录结果。帮助维护人员判断问题停留在本地、网关、运营商还是目标侧。

• 自动判断断在哪一跳 • Windows 用 tracert，Linux 用 traceroute • 支持本地文件留痕与服务端汇总
🌐

多 DNS 延迟探测

同时测量百度、阿里、腾讯、网易四组 DNS 响应，帮助你区分“网络不可达”和“DNS 质量下降”这两类完全不同的问题。

• 百度 / 阿里 / 腾讯 / 网易 四组延迟 • 卡片化展示，手机端也易读 • 更适合判断运营商和 DNS 质量
🗺️

静默 ARP 拓扑发现

通过读取本机 ARP 缓存发现局域网设备，不主动扫描整个网段。更安静、更轻量，也更适合在真实办公网络中长期运行。

• 不主动扫 254 个 IP • 自动识别 IP / MAC / hostname / 厂商 • 多客户端汇总后覆盖全网
使用场景

把“客户说网络有问题”变成一组可以快速判断的数据。

对维护方来说，最重要的不是看到更多图，而是在更短时间内判断责任边界、沟通方向和下一步动作。

传统方式
客户反馈“网络异常”，后台通常只能显示设备离线。
可以看到现象，但很难判断故障停留在哪一段链路。
ICMP 被禁时容易误判，维护沟通成本高。
依赖全网 ping 的拓扑扫描，对办公网络并不友好。
LANWatch
DNS、ICMP 与目标状态同时展示，先快速确认问题范围。
离线后自动触发 traceroute，保留“断在哪一跳”的诊断结果。
ICMP 不支持时明确标注，避免把协议限制误读为网络故障。
ARP 静默发现更适合长期部署，不打扰客户网络环境。

接入方式

从客户端起步，再扩展到 SNMP，部署路径足够轻也足够稳。

LANWatch 适合先从最容易落地的客户端安装开始，再根据网络规模补充企业级设备的 SNMP 接入。

客户端安装

适合办公室电脑、值守主机以及日常运行中的 Windows / Linux 终端。部署门槛低，几分钟内即可开始采集。

• 下载后直接运行，填写企业名称完成注册 • 默认后台运行，支持开机自启 • 自动上报探测数据、自动进行离线诊断

SNMP 接入

适合企业级路由器、交换机、防火墙等无法安装 Agent 的设备，作为客户端监控之外的补充接入层。

• 支持 SNMP V2c • 默认 community: public • 后续可扩展更多企业级设备模板
