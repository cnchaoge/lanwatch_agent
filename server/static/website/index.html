核心能力

三个能力，组合成一套完整值守体系。

LANWatch 不只是告诉你"断了"，而是告诉你"断在哪"以及"为什么断"。

🔍

离线断点诊断

连续 3 次探测失败后，自动触发 traceroute，定位断点在第几跳。不依赖现场，不依赖客户描述。

  • 自动判断断在本地 / 路由器 / 运营商 / 目标侧
  • Windows 用 tracert，Linux 用 traceroute
  • 结果本地留档，同时上报服务端
  • 手机端直接查看诊断结论
🌐

多 DNS 延迟探测

同时测百度、阿里、腾讯、网易四组 DNS 响应。不只测"通不通"，更测"哪家 DNS 变慢了"。

  • 四个域名同时探测，更真实反映 DNS 质量
  • 帮助区分"DNS 问题"和"网络不可达"
  • 卡片化展示，手机端同样清晰
  • 历史数据支撑运维决策
🗺️

静默 ARP 拓扑发现

直接读取本机 ARP 缓存表，秒级完成。零噪音，不发包，不触发防火墙，不打扰客户网络。

  • 不发任何 ICMP 包，无入侵性
  • 自动识别 IP / MAC / hostname / 厂商
  • 多客户端分布在不同网段时，汇总即覆盖全网
  • 适合长期在真实办公网络运行

使用场景

客户说"断网了"，你怎么回答？

传统监控只显示"离线"。LANWatch 显示"断在哪一跳"。这是本质区别。

传统方式
  • 客户报修"断网了"，你打开后台 → 只看到设备离线，不知道断在哪。
  • ICMP 被路由器禁了 → 误报频繁，解释成本高，维护体验差。
  • 网络拓扑靠全网 ping → 慢、吵，容易触发防火墙，客户反感。
  • DNS 慢的问题 → 不知道是运营商问题还是 DNS 服务商问题。
LANWatch
  • 设备离线 → 立即看到"上次离线原因：第 3 跳超时（ISP 出口）"，快速定位责任方。
  • ICMP 不支持 → 后台明确标注"不支持"，不误判，不混乱。
  • ARP 静默发现 → 秒读本机缓存，不发包，不打扰客户网络。
  • 多 DNS 探测 → 同时看到四家 DNS 响应时间，区分"运营商慢"还是"DNS 故障"。
