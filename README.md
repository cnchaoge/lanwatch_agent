# lanwatch_agent - 企业网络值守服务

> 让每一家中小企业的网络都有"人"值守

[![版本](https://img.shields.io/badge/version-v0.6.0-blue.svg)](https://github.com/cnchaoge/lanwatch_agent/releases)
[![平台](https://img.shields.io/badge/platform-Windows%20%7C%20Linux%20%7C%20macOS-green.svg)]()
[![协议](https://img.shields.io/badge/license-MIT-orange.svg)]()

**lanwatch_agent** 是一套面向中小企业网络监控的 SaaS 解决方案，包含运行在客户内网的监控客户端（Agent）和部署在云端的服务端。客户无需专业人员，下载运行即可完成部署，管理员和企业主通过 Web 后台或手机随时掌握网络健康状态。

---

## 🎯 解决什么问题

三四线城市的中小企业普遍面临网络故障发现滞后的问题——往往是业务受损后才报修，运维响应被动。**lanwatch_agent** 将监控能力打包成"网络值守服务"：企业安装一个 exe，服务端持续采集数据，离线即报警，手机随时查看。

## 🎯 核心亮点：离线断点诊断

> 传统监控只告诉你"网络断了"，lanwatch 告诉你"断在哪一跳"

**场景：** 客户打电话说"断网了"，你打开后台看到设备离线，但不知道断在哪。

**lanwatch 的做法：** 客户端检测到连续 3 次探测失败 → 自动触发 traceroute → 结果保存本地并上报服务端 → 客户打开客户端直接看到"上次离线原因：第 3 跳超时（ISP出口）"

**客户感知：** 断网后打开客户端 → 直接看到断在哪 → 节省大量沟通和现场排查时间

**技术细节：**
- 连续 3 次探测失败触发诊断
- traceroute 结果本地保存 `~/.lanwatch_offline_diag.txt`
- 可选上报服务端，管理后台展示每台设备最近离线原因
- Windows 用 `tracert`，Linux 用 `traceroute`

## ⚡ 技术亮点：多协议智能探测

> 为什么能适配各种网络环境？因为不依赖单一协议

### 传统方案的问题：单协议探测

很多路由器、企业网关禁用了 ICMP ping，导致大量假离线：
- ICMP ping → 被防火墙拦截 → ❌ 误报网关离线
- TCP 80/443 探测 → 依赖 HTTP 服务开着 → ❌ 打印机/摄像头/工业设备全漏报

### lanwatch 的方案：多协议兜底

```
探测优先级：ICMP → TCP 80/443 → DNS
哪个通算哪个，不依赖单一协议
```

```python
# 1. 先试 ICMP ping
ok = ping_icmp(host)
if ok:
    return "ICMP OK", rtt

# 2. ICMP 不通？试 TCP 握手
ok = ping_tcp(host, port=443)
if ok:
    return "TCP OK", rtt

# 3. TCP 也不通？试 DNS 解析
ok = ping_dns(host)
if ok:
    return "DNS OK", None
```

**实测效果：**

| 网络环境 | 旧方案（纯ICMP） | lanwatch 多协议 |
|----------|------------------|-----------------|
| ICMP 通的路由器 | ✅ | ✅ |
| 禁 ICMP 的路由器 | ❌ 假离线 | ✅ TCP 兜底 |
| DNS 不通但 TCP 通 | ❌ 假离线 | ✅ |
| 复杂企业网络 | ❌ 频繁误报 | ✅ 自动适配 |

### 拓扑扫描性能

| 方案 | 254 IP 扫描耗时 | CPU 利用率 |
|------|----------------|-----------|
| 串行探测 | ~85 秒 | 低 |
| **多线程并发（30 workers）** | **~5 秒** | 高效 |

v0.6 采用 `ThreadPoolExecutor(max_workers=30)` 并发探测，充分利用网络等待间隙，254 个 IP 从 85 秒缩短到 5 秒以内。

## 📦 安装部署

### 企业主（客户端）

**Windows 版本（推荐）：**

1. 下载 `lanwatch_agent.exe`：https://github.com/cnchaoge/lanwatch_agent/releases/latest
2. 双击运行，填写企业名称
3. 首次运行后可勾选「开机自动启动」
4. 最小化到托盘，开始监控

**服务端地址：** `http://82.156.229.67:8000`

**手机查看：** http://82.156.229.67:8000/mobile

**下载中心：** http://82.156.229.67:8000/download

---

### 开发者（从源码构建）

**依赖：** Python 3.8+

```bash
# 克隆仓库
git clone https://github.com/cnchaoge/lanwatch_agent.git
cd lanwatch_agent/agent

# 安装依赖
pip install pystray pillow pyinstaller

# 打包 Windows exe（需要 Windows 系统）
cd windows && build.bat

# 生成的 exe 在 dist/lanwatch_agent.exe
```

---

### 私有部署（服务端）

服务端依赖 Python 3.8+ 和 FastAPI：

```bash
cd server
pip install -r requirements.txt
python main.py
# 服务监听 0.0.0.0:8000
```
